Diagnóstico de Conexão Brisanet

Aprenda a usar Ping, Tracert e Pathping para encontrar a causa de lentidão e perda de pacotes.

Como Realizar os Testes Interpretando os Resultados

Entendendo as Ferramentas de Diagnóstico

Cada teste serve para investigar um aspecto diferente da conexão.

O Caminho da sua Conexão

Quando você acessa um site ou joga online, seus dados passam por vários roteadores (saltos) até chegar ao destino. Problemas podem ocorrer em qualquer ponto desse caminho.

  • Ping: Mede o tempo de ida e volta até o destino final. É um teste rápido para ver a latência (o "lag").
  • Tracert (Traceroute): Mostra cada "salto" no caminho e o tempo para chegar em cada um. Ajuda a mapear a rota.
  • Pathping: É o mais completo. Ele testa a rota como o Tracert e depois mede a perda de pacotes em cada salto por um tempo, sendo o mais preciso para encontrar problemas.

Como Realizar os Testes

Os testes são feitos no Prompt de Comando (CMD) do Windows.

Passo 1: Abrir o Prompt de Comando (CMD)
Pressione a tecla Windows, digite cmd e aperte Enter. Uma tela preta irá aparecer.

Comando Ping

Use este comando para um teste contínuo de latência. Para testar o servidor do Google, por exemplo, digite:

ping 8.8.8.8 -t

Pressione Ctrl + C para parar o teste.

O que observar: O valor em "tempo=". Quanto menor, melhor. Se aparecer "Esgotado o tempo limite do pedido", significa que um pacote de dados se perdeu no caminho.

Comando Tracert

Este comando mapeia a rota até o destino. Para testar, digite:

tracert 8.8.8.8

Aguarde o teste terminar (ele mostra "Rastreamento concluído").

O que observar: A lista de IPs (saltos) e os tempos (em ms) para cada um. Observe se há um salto repentino na latência.

Comando Pathping

Este é o teste mais detalhado e recomendado para diagnosticar perda de pacotes. Digite:

pathping 8.8.8.8

Atenção: Este teste leva alguns minutos para ser concluído. Aguarde até ele mostrar a tabela final de estatísticas.

O que observar: A coluna "Perdido/Enviado = Pct". Se algum salto mostrar uma porcentagem maior que 0%, ali está um ponto de perda de pacotes.

Interpretando os Resultados: Onde está o Problema?

Com o resultado do Tracert ou Pathping em mãos, vamos identificar a origem do problema.

Cenário 1: Problema na Rede da Brisanet

Como identificar: A latência (ping) sobe muito ou a perda de pacotes (Pathping) começa em um dos primeiros saltos da rota. Na Brisanet, esses saltos são:

  • Seu gateway, que geralmente tem um IP na faixa 172.16.x.x.
  • Os primeiros IPs públicos logo após essa faixa.

✅ Ação a Tomar:

O cliente entra em contato com o suporte. solicite uma captura de tela ou o texto completo dos testes Pathping. Estando confirmado que o problema esta na rede interna e verifique os proximos passos com a liderança ou o setor NOC.

Cenário 2: Problema na Rota Externa

Como identificar: Os primeiros saltos na rede da Brisanet estão normais (ping baixo, 0% de perda), mas a latência sobe ou a perda de pacotes começa em um salto no meio do caminho, em um IP que não pertence à Brisanet.

✅ Ação a Tomar:

Neste caso, a Brisanet não tem controle sobre o roteador de outra empresa. A única solução viável é solicitar a contratação de um IP Fixo. Com um IP Fixo, você sai da rota compartilhada dos IPs dinâmicos e recebe uma nova rota, que geralmente é mais direta e estável, resolvendo o problema.

Cenário 3: Problema no Servidor de Destino

Como identificar: Todos os saltos no caminho têm latência boa e 0% de perda de pacotes, mas o ping fica alto ou há perda de pacotes apenas no último salto, que é o IP do servidor do jogo ou site.

✅ Ação a Tomar:

Infelizmente, não há nada que você ou a Brisanet possam fazer. O problema está no servidor de destino. A solução é aguardar que os administradores do servidor (do jogo, site, etc.) resolvam o problema ou tentar usar um servidor diferente, se houver essa opção.

Perguntas Frequentes

Respostas para dúvidas comuns sobre diagnóstico de rede.

Significa que aquele roteador (salto) não respondeu ao teste. Se isso acontecer em um ou dois saltos no meio, mas o teste continuar e chegar ao destino, não é um problema. É apenas um roteador configurado para não responder a esse tipo de solicitação. O problema real é quando os asteriscos continuam até o final do teste.

Clientes com IP dinâmico compartilham grandes blocos de rotas. Mudar a rota para um cliente significaria mudar para centenas ou milhares de outros clientes que usam o mesmo caminho. Isso é tecnicamente inviável e poderia causar instabilidade e conflitos de IP na rede. Por isso, a solução para um problema de rota é obter uma rota exclusiva e diferente através do IP Fixo.

Um IP Fixo resolve a maioria dos problemas relacionados a rotas ruins ou congestionadas e elimina o CGNAT, o que melhora drasticamente a estabilidade e a qualidade da conexão para jogos e outras aplicações. No entanto, ele não pode corrigir um problema que está no servidor de destino. Se o servidor do jogo estiver sobrecarregado ou for muito distante geograficamente, o ping ainda pode ser alto.
